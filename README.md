# منصة Elite International Schools (مدارس النخبة العالمية)

نموذج كامل لمنصة تعليمية تشمل:
- **واجهة ولي الأمر**: الواجبات، الحصص الأونلاين، الحضور، التقارير اليومية، الدرجات، الجدول، الاختبارات، الأنشطة والصور، الرسائل، ملف الطفل، وزر تواصل مباشر عبر واتساب مع الإدارة.
- **لوحة الإدارة**: إدارة الطلاب وأولياء الأمور، الواجبات، الحضور، التقارير، الدرجات، الجدول، الاختبارات، رفع صور الأنشطة، الإعلانات، الرسائل، وإعدادات واتساب.

## ☁️ التخزين السحابي

- **Firebase Firestore**: كل بيانات الطلاب، الواجبات، الحضور، التقارير، الدرجات، الاختبارات، الرسائل، الإعدادات. مساحة مجانية: **1 جيجابايت** + 50,000 قراءة و20,000 كتابة يوميًا.
- **Cloudinary**: صور الأنشطة المرفوعة من لوحة الإدارة. مساحة مجانية: **25 جيجابايت**.

> 💡 لماذا Cloudinary وليس Firebase Storage؟ لأن تفعيل Firebase Storage يتطلب ترقية المشروع لخطة Blaze (الدفع حسب الاستخدام)، وفي بعض الدول (مثل السعودية) هذا يتطلب المرور ببوابة فوترة محلية (CNTXT) لا تدعم حاليًا تسجيل الأفراد. Cloudinary بديل مجاني تمامًا وبدون أي بطاقة بنكية.

أي تعديل من الإدارة (واجب جديد، تقرير، درجة، حضور، صورة...) **يظهر فورًا** عند ولي الأمر على أي جهاز/متصفح، والعكس صحيح (مثل الرسائل)، لأن البيانات متزامنة لحظيًا (Real-time) عبر Firestore.

## 🚀 خطوة وحيدة متبقية: تفعيل قواعد أمان Firestore

عند إنشاء قاعدة بيانات Firestore في "Production mode"، تكون القواعد الافتراضية **"رفض الكل"** (لا قراءة ولا كتابة لأي شخص، حتى التطبيق نفسه). لتشغيل المنصة:

1. روح على [console.firebase.google.com](https://console.firebase.google.com) → مشروع **eis-portal**
2. ☰ → **Security** → **Firestore Database** → تبويب **"Rules"**
3. استبدل المحتوى بالكامل بالموجود في ملف **`firestore.rules`** المرفق مع المشروع:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /{document=**} {
         allow read, write: if request.auth != null;
       }
     }
   }
   ```
4. اضغط **"Publish"**

> ⚠️ **ملاحظة أمان**: هذه القاعدة تتطلب فقط أن يكون المستخدم "مسجّل دخول كزائر" (يحدث تلقائيًا عند فتح الموقع، لأن Anonymous Authentication مفعّلة). هذا يمنع بوتات الإنترنت العشوائية، لكنه **لا يفرّق بين الإدارة وولي الأمر** على مستوى قاعدة البيانات — أي شخص يفتح الموقع يمكنه نظريًا قراءة/تعديل كل البيانات بما فيها كلمات المرور. هذا النموذج **مناسب كنسخة تجريبية/داخلية لمدرسة موثوقة**، وللاستخدام الفعلي مع بيانات أطفال حساسة (PDPL) يُنصح بشدة بالاستعانة بمطور لإضافة:
> - نظام مصادقة حقيقي (Firebase Authentication بالجوال/OTP).
> - قواعد أمان مبنية على الأدوار (admin / parent).

## 🔧 البناء والتشغيل

```bash
npm install
npm run build
```
سيظهر مجلد **`dist`** — هذا ما يُرفع على Netlify.

للتجربة محليًا:
```bash
npm run dev
```
يفتح على `http://localhost:5173`.

---

## 🌐 الرفع على Netlify

### الطريقة 1 — Netlify Drop (الأسرع)
1. نفّذ `npm install && npm run build` (ملف `.env` المرفق سيُستخدم تلقائيًا)
2. اذهب إلى https://app.netlify.com/drop واسحب مجلد **`dist`** فقط (وليس المشروع كامل أو ملف مضغوط)

### الطريقة 2 — ربط مع Git (تحديثات تلقائية)
1. ارفع المشروع (بدون `node_modules`, `dist`, `.env`) إلى GitHub
2. Netlify: "Add new site" → "Import from Git" → Build command: `npm run build` | Publish directory: `dist`
3. **مهم**: في "Site settings → Environment variables" أضف كل المتغيرات الثمانية من `.env` (لأن `.env` لا يُرفع لـ Git)
